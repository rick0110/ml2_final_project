                        TTS PIPELINE
        ┌─────────────────────────────────────────┐
        │                                         │
TEXT    │  Text  →  FastPitch  →  h_text         │  (frozen)
INPUT   │ Encoder    (384 dims)                  │
        │                                         │
        │  Mel    →    GST     →  z_style        │  (trainable)
TARGET  │ Input     (128 dims)                   │
MEL     │                                         │
        │  Concatenate: [h_text, z_style]        │
        │         ↓                              │
        │  Acoustic Decoder (LSTM)  ← TRAINABLE │
        │         ↓                              │
        │   M_hat (80 mel bins)                  │
        │         ↓                              │
        │   HiFi-GAN Vocoder   ← frozen          │
        │         ↓                              │
OUTPUT  │   x_hat(t) Audio                       │
        │                                         │
        └─────────────────────────────────────────┘

================================================================================
                            LOSS FUNCTIONS
================================================================================

1. L1 Reconstruction Loss
   L_recon = |M_predicted - M_target|_1
   Minimizes absolute difference between mel spectrograms

2. Style Diversity Loss
   L_div = mean(ReLU(cos_similarity - (1 - margin)))
   Penalizes style embeddings from collapsing into same space

3. Total Loss
   L_total = w_recon * L_recon + w_diversity * L_div
   Default weights: w_recon=1.0, w_diversity=0.5

================================================================================
                          OUTPUT STRUCTURE
================================================================================

experiments/step_1/
└── attempt_YYYYMMDD_HHMMSS/  (auto-created with timestamp)
    ├── config.json            (all hyperparameters saved)
    ├── checkpoints/           (model snapshots)
    │   ├── epoch_0001.pt      (after epoch 1)
    │   ├── epoch_0002.pt      (after epoch 2)
    │   ├── ...
    │   └── best.pt            (best by validation loss)
    ├── tensorboard/           (TensorBoard event files)
    │   └── events.out.tfevents.XXXXX
    └── logs/                  (optional text logs)

Each checkpoint contains:
  - Model weights
  - Optimizer state
  - Epoch number
  - Training metrics (losses, etc.)
