\subsection{Directory Roles}
\begin{itemize}[leftmargin=1.5em]
    \item \textbf{\pathtt{data/}:} The lifecycle of audio starts in \pathtt{data/raw/} and transitions to \pathtt{data/processed/} via the preprocessing scripts.
    \item \textbf{\pathtt{scripts/}:} Houses standalone execution scripts. Preprocessing scripts iterate over raw corpora, standardize sample rates, extract initial acoustic features, and write manifest CSVs.
    \item \textbf{\pathtt{src/models/}:} The core PyTorch implementation. Divided explicitly into \pathtt{tacotron2\_vae} (handling text-to-mel logic) and \pathtt{waveglow} (handling mel-to-audio logic).
    \item \textbf{\pathtt{src/training/}:} Contains isolated training loops. Tacotron uses a standard iteration-based loop, while WaveGlow implements Distributed Data Parallel (DDP).
\end{itemize}

\section{Data Pipeline and Preprocessing}

The data pipeline bridges the gap between raw datasets and PyTorch \pathtt{DataLoader} iterations. The pipeline consists of two primary phases: off-line preprocessing (\pathtt{scripts/preprocess}) and on-line data loading (\pathtt{src/data}).

\subsection{Phase 1: Offline Preprocessing}
The repository provides specialized scripts for different datasets (e.g., \pathtt{preprocess\_libri-Speech-en\_vaeTacotron.py}). 

\textbf{Workflow sequence:}
\begin{enumerate}
    \item \textbf{Discovery:} The script recursively glob-searches for \pathtt{*.trans.txt} files.
    \item \textbf{Text Loading:} Each row matches an utterance ID to a transcription text.
    \item \textbf{Audio Normalization:} The corresponding audio file is loaded via \pathtt{torchaudio}. It is converted to mono (by averaging channels if necessary) and resampled to the target sampling rate (16000 Hz for EN, 22050 Hz for PT).
    \item \textbf{Feature Extraction:} A specialized transform computes the 80-bin Mel-spectrogram. For Tacotron2-VAE compatibility, the \pathtt{Tacotron2MelTransform} utilizes \pathtt{librosa.filters.mel} to guarantee parity with the original Tacotron codebase.
    \item \textbf{Duration Filtering:} Utterances shorter than 0.3s or longer than 10.0s (to prevent OOM errors) are discarded.
    \item \textbf{Serialization:} The waveform, mel-spectrogram, text, duration, and sampling rate are serialized into a \pathtt{.pt} file.
    \item \textbf{Manifest Generation:} A \pathtt{mels\_metadata.csv} is generated containing \pathtt{mel\_path, duration, text, utt\_id}.
\end{enumerate}

\textbf{Preprocessing Hyperparameters (LibriSpeech-EN):}

\subsection{Phase 2: Online Dataset Loading}
In \pathtt{src/data/loader\_vae\_tacotron/loader\_tacotron.py}, the \pathtt{DatasetLibriSpeechTacotronVAE} class reads the generated \pathtt{mels\_metadata.csv}.

\textbf{Crucial Implementation Detail:}
Although the offline scripts extract Mel-spectrograms, the \pathtt{loader\_tacotron.py} \textit{ignores} the serialized Mel-spectrogram and re-computes it on the fly during the \pathtt{\_\_getitem\_\_} call. 
\begin{lstlisting}[language=Python]
# Excerpt from loader_tacotron.py
def get_mel(self, audio):
    audio_norm = audio / MAX_WAV_VALUE
    audio_norm = audio_norm.unsqueeze(0)
    audio_norm = torch.autograd.Variable(audio_norm, requires_grad=False)
    melspec = self.stft.mel_spectrogram(audio_norm)
    return torch.squeeze(melspec, 0)
\end{lstlisting}

\textbf{Online STFT Configuration:} The online STFT relies on \pathtt{TacotronSTFT} defined in \pathtt{layers.py}, configured specifically with \pathtt{filter\_length=800, hop\_length=200, win\_length=800, sampling\_rate=22050}. 

\textbf{Text Normalization:} The text is passed through \pathtt{TextNormalizerEN}, which utilizes Regex to strip punctuation and the \pathtt{num2words} library to spell out integer values before tokenization.

\textbf{Emotion Tensor Injection:} An emotion tensor of shape \pathtt{[4]} is initialized and attached to every batch. By default, index 0 is set to 1.0 (Neutral), acting as a placeholder for later fine-tuning on affective datasets.

\section{Model Architectures}

The repository generates audio via a two-stage cascaded pipeline. The diagrams below have been structured to ensure readability and prevent module overlap.

\subsection{5.1 Complete System Overview}

Text transcripts are converted to character sequences and fed into \textbf{Tacotron2-VAE}. Simultaneously, the target audio's mel-spectrogram is ingested by the VAE module to extract a latent prosody embedding. Tacotron predicts a highly expressive Mel-spectrogram, which is subsequently inverted into a 1D waveform by \textbf{WaveGlow}.

\subsection{5.4 VAE (Prosody) Breakdown}

Instead, it is a pure Variational Autoencoder that maps the reference mel to a latent distribution, samples $z$, and projects it linearly to a style embedding.

\section{Loss Functions}

The models are trained independently using custom PyTorch criteria defined directly in their respective submodules.

\subsection{Tacotron2-VAE Losses (\pathtt{losses.py})}

The \pathtt{Tacotron2LossVAE} class evaluates three distinct objectives.

\subsubsection{1. Reconstruction Loss ($\mathcal{L}_{\text{Recon}}$)}
Mean Squared Error is computed both before and after the PostNet to ensure the recurrent network learns the gross spectral envelope while the PostNet refines the residual details.
\[
\mathcal{L}_{\text{Recon}} = \| \text{Mel}_{\text{target}} - \text{Mel}_{\text{decoder}} \|_2^2 + \| \text{Mel}_{\text{target}} - \text{Mel}_{\text{postnet}} \|_2^2
\]

\subsubsection{2. Gate Loss ($\mathcal{L}_{\text{Gate}}$)}
Binary Cross Entropy with Logits penalizes incorrect predictions of the stop token.
\[
\mathcal{L}_{\text{Gate}} = \text{BCEWithLogits}(gate_{\text{predicted}}, gate_{\text{target}})
\]

\subsubsection{3. Kullback-Leibler Divergence ($\mathcal{L}_{\text{KL}}$)}
Regularizes the latent space $z$ to approximate a standard normal distribution $\mathcal{N}(0, I)$.
\[
\mathcal{L}_{\text{KL}} = -0.5 \sum_{i} \left( 1 + \log(\sigma_i^2) - \mu_i^2 - \sigma_i^2 \right)
\]

\subsubsection{4. KL Annealing Strategy}
To prevent posterior collapse, a dynamic weight $w_{KL}$ scales the KL loss. The repository allows \texttt{logistic}, \texttt{linear}, or \texttt{constant} scheduling. The default is logistic:
\[
w_{KL}(step) = \frac{\text{upper}}{\text{upper} + \exp(-k \times (step - x_0))}
\]
Where $\text{upper}=0.2$, $k=0.0025$, and $x_0=10000$.

\textbf{Total Tacotron Objective:}
\[
\mathcal{L}_{\text{Tacotron}} = \mathcal{L}_{\text{Recon}} + \mathcal{L}_{\text{Gate}} + w_{KL} \cdot \mathcal{L}_{\text{KL}}
\]

\subsection{WaveGlow Losses (\pathtt{glow.py})}

WaveGlow is trained to maximize the exact log-likelihood of the data. Because it uses normalizing flows, the objective relies on the change of variables theorem. The loss is effectively the Negative Log-Likelihood (NLL).

The \pathtt{WaveGlowLoss} class computes:
\[
\mathcal{L}_{\text{WaveGlow}} = \frac{1}{B \cdot C \cdot T} \left( \sum_{i} \frac{z_i^2}{2\sigma^2} - \sum \log s - \sum \log |\det W| \right)
\]
Where:
\begin{itemize}[leftmargin=1.5em]
    \item $z$ is the latent representation mapped back to the prior.
    \item $\sigma$ regulates the variance penalty (default $1.0$).
    \item $\log s$ is the sum of scale logs generated by the affine coupling layers.
    \item $\log |\det W|$ is the log determinant of the invertible $1 \times 1$ convolutions.
    \item Normalization relies on Batch size ($B$), Channels ($C$), and Time length ($T$).
\end{itemize}

\section{Training Pipeline}

Training is orchestrated through execution scripts within \pathtt{src/training/}.

\subsection{Tacotron2-VAE Training}
The training entrypoint is \pathtt{src/training/training-tacotron2-vae/train.py}.
\begin{itemize}[leftmargin=1.5em]
    \item \textbf{Setup:} Reads the \texttt{mels\_metadata.csv} manifest, builds character symbols, and instantiates \pathtt{Tacotron2VAEHparams}.
    \item \textbf{DataLoaders:} Instantiates \pathtt{DatasetLibriSpeechTacotronVAE} and uses \pathtt{TextMelCollate} to pad batches dynamically. Note that audio waveforms are loaded and converted to Mels iteratively during fetching.
    \item \textbf{Optimizer:} Standard PyTorch \pathtt{Adam} optimizer.
    \item \textbf{Loop:} Operates over fixed \pathtt{epochs}. Gradient clipping is applied (\pathtt{thresh=1.0}). Checkpoints are saved every \pathtt{iters\_per\_checkpoint} interval.
\end{itemize}

\textbf{Execution Example:}
\begin{lstlisting}[language=bash]
python src/training/training-tacotron2-vae/train.py \
    --epochs 300 \
    --batch-size 32 \
    --learning-rate 1e-3 \
    --artifacts-dir data/processed/libriSpeech-en-tacotron-vae \
    --experiment-name v1_tacotron
\end{lstlisting}

\subsection{WaveGlow Training}
The training entrypoint is \pathtt{src/training/training-waveglow/train.py}.
\begin{itemize}[leftmargin=1.5em]
    \item \textbf{Setup:} Parses a \texttt{config.json} containing \texttt{data\_config}, \texttt{dist\_config}, and \texttt{waveglow\_config}.
    \item \textbf{Distributed Training:} Heavily leverages PyTorch Distributed Data Parallel (DDP). Initializes process groups across available GPUs.
    \item \textbf{Mixed Precision:} Utilizes NVIDIA Apex (\pathtt{amp.initialize}) with \texttt{O1} optimization to drastically reduce VRAM overhead.
    \item \textbf{Loop:} Reads precomputed mel-spectrograms from disks, passes them through the reverse flow, computes the NLL loss, and applies \pathtt{reduce\_tensor} across GPUs.
\end{itemize}

\textbf{Execution Example:}
\begin{lstlisting}[language=bash]
python -m torch.distributed.launch --nproc_per_node=2 \
    src/training/training-waveglow/train.py \
    -c config.json \
    --checkpoints_dir experiments/waveglow/checkpoints
\end{lstlisting}
